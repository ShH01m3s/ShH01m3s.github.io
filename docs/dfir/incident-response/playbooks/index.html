IR Playbooks

domain: forensics

doctype: article


These are my ideas on IR playbooks for training. Don’t judge :) We don’t evolve if we don’t make mistakes and learn from them. And we don’t make mistakes if we do nothing. Even if I am risking to become an object of ridicule, I am posting this article. I am planning to imporve once I learn more or gain some relevant experience.

Data Exfiltration

Preparation 🔪🥒

  • Identify confidential information and limit the amount of people allowed to work with it. Remember the least privilege principle.
  • Force people to sign NDA, consult with the legal department on the best ways to do so.
  • Deploy some automated attachment scanning. For example, place some magic value in the metadat of all documents that contain confidential data. This will not protect against techies though.
  • For every employee there must be a digital signature, which is applied to the document each time it’s created or copied. Force every e-document to be signed with this signature.
  • Restrict copying documents to the external media, such as USB drives (this sort of security mechanism was deployes at Allianz some time ago, not sure about now).
  • Support high loyalty and corporate culture. People who like their company and job, who feel appreciated and well-paid, will have less temptation to sell the information, I presume. If you exfiltrate some data and this data will decrease the revenue of the company, you might have no bonus.
  • Configure and IDS so that it will inspect emails to external addresses that have no BCC and CC. It’s highly unlikely that a person exfiltrating some data will add a person in a CC field.

NIST Detection (SANS Identification) 🚨

  • External parties might inform on data exfiltration or another company has announced some product that was originally invented by this company.
  • Alerts received from some IDS.

NIST Analysis (SANS Identification) 🔬 🧪 🔎

  • When some data is known to be exfiltrated, refer to your policy, that states who has the access to this data.
  • If you have the e-document iteslf and it’s digitally signed, then you have some point to start.
  • Review the logs to see, when the oriiginal document in question was last opened and modified and by whom. Interrogate the person.
  • Review the network traffic to determine which machine is responsible for this leak (not neccessarilly the person using it).
  • Check the network traffic for other possible leaks.
  • Check the host for possible malware intrusion or hacking activity.
  • Im case it’s a Mac, check AirDrop logs as well.

Containment 🪤

  • Isolate the host that’s suspected to be leaking the information.
  • Restrict the access to confidential information.
  • Generate new digital certificate for the person whose DS was compromised (that is if this person is not guilty, of course).

Eradiction 🌱

  • Remove the malware (if found).
  • Reimage the host (if neccessary). For example, if the malware was not found but it’s known to be there, from the IR perspective you are supposed to act fast, no time for exploring and learning. Just get the machine back to the working state as soon as possible.

Recovery 💊

  • Scan the host for vulnerabilities.
  • Monitor it for anomalous traffic again.
  • Review the access policy and IDS rules.

NIST Post-incident Activity (SANS Lessons Learned) 📚

  • Think about what was wrong. May be you were not quick enough.

Social Engineering

Below is an example from the book [2].

Preparation 🥒

  • Train the employee to detect suspocious emails and forward them to the security team.
  • Deploy some automated attachment scanning.

NIST Detection (SANS Identification) 🚨

  • Employees forward suspicios emails.
  • Alerts received from some AV or HIPS.

NIST Analysis (SANS Identification) 🔬 🧪 🔎

  • Examine the compromised host memory.
  • Examine the event logs for suspicious entries.
  • Examine the network traffic going to and from the host.

Containment 🪤

  • Isolate the host if it was proved to be under attack.

Eradiction 🌱

  • Remove the malware identified.
  • Reimage the host (if neccessary). For example, if the malware was not found but it’s known to be there, from the IR perspective you are supposed to act fast, no time for exploring and learning. Just get the machine back to the working state as soon as possible.

Recovery 💊

  • Scan the host for vulnerabilities.
  • Monitor it for anomalous traffic.

NIST Post-incident Activity (SANS Lessons Learned) 📚

  • Think about what was wrong. May be you were not quick enough.
