How to prove compromise❓

domain: forensics

doctype: article


Actually, as I read further an further, it seems that everything is about a user. You just cannot do something without using SOME user. But!

Below are some thoughts regarding conducting an investigation. A user can be of several categories, as I see it. I’ll name each one as shortly as I can to refer to them in future. But I will describe them in more detail below.

Legitimate user, who has done something bad by accident. Let’s call him a dummy 😬. I’ve been there, I once changed permissions to Deny for my admin account to disk C on Windows and could do nothing about it later. How stupid was it 🙈? My excuse is that I only started with PCs… . Another example is becoming a victim to social engineering. This type can lead directly to the last one 🦓 (see below). Also can have 🦠 malware involved. It doesn’t always mean a person is in fact a dummy, especially in case of very sofisticated attacks.

Legitimate user, who has done something bad by intent. Let’s call this one a poopy 💩. Say, he or she feels highy underpaid and sells company’s kitty plans to some evil dog hackers. This is usually about data exfiltration. They might willingly plant some 🦠 malware as well.

A user created and used by a hacker. This one I will call a sheep 🐑 as in A wolf in sheep’s clothing, since the attacker is trying mask itself as something good. This one is not unusual, but I guess, easily determined by the sysadmin. However, there are some cases, when an attacker can change view permissions for his/her account on AD so that no one will see this user. This is also a type of user which sometimes would require a 🦠 malware to be installed.

A user created legitimately, but compromised by a hacker and doing something by intent. This one will be a donkey 🦓, as the attaker is using another account for his ride. I guess it’s the most common when the hackers are in the picture. Also, this the main type where 🦠 malware is in the picture. This one must involve either social engineering, or leveraging some vulnerabilty in the network.

❓ So, back to the question. How to prove that the user account was compromised?

Lot’s of attacks involve account compromise. In some cases malware is not even needed. There are several arguments for the defence: I didn’t know, It was not me, It was a malware and It happened accidently.

✍️ Suspicious logins?

  • Should this user use this account?
  • Are they new to the computer?
  • If remote, does the account usually come from that computer?
  • Is the time expected?
  • Does the account have lots of failed logins? 👉 A very strong indicator of compromise
  • A login during time when the user was absent 👉 A very strong indicator of compromise

✍️ Malware. There is a separate section for malware above ☝️, but it can be a strong defense for some one claiming his or her account was compromised. But! ⚠️ Even if there is a malware found, it’s essential to prove that it existed at the time that some investigated activity took place.
