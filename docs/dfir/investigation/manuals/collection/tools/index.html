🧰 Acquisition Toolkit

   

Validate Tools

Document the changes that you are making. Know your tools 🛠! What changes they are making (see).

🧰 Validate your tools. Tools used need to be periodically validated prior to each analysis. For example, check that you wiping software has thoroughly WIPED the drive you will be using for the evidence collectionc (for example, by running a checksum64 against it).

Choose the appropriate one. Below are some questions to ask in order to decide.

Example 1. Logging in remotely to collect artifacts and run preinstalled UI admin tools:

Question Answer
Easy to collect artifacts? Hard 🧱: Many non-DFIR tools
Easy to connect remotely? Easy 💸: if remote connection is allowed
Identify relevant data? Hard 🧱: not DFIR specific
Pivoting Hard 🍯-🧱: Depends
Scaling Hard 🧱: one at a time

Example 2. Logging in remotely to collect artifacts with cmd tools:

Question Answer
Easy to collect artifacts? Medium 🍯. Many tools, but can be scripted. Generally free.
Easy to connect remotely? Hard 🧱: have to copy tools to the system.
Identify relevant data? Hard 🧱: not DFIR specific
Pivoting Hard 🧱: Many output files
Scaling Hard 🧱: one at a time

Example 3. Continuous monitoring:

Question Answer
Easy to collect artifacts? Easy 💸: for artifacts being monitored. Hard 🧱: for others.
Easy to connect remotely? Easy 💸: if configured.
Identify relevant data? Medium 🍯: depends on the tools.
Pivoting Medium 🍯: depends on the tools.
Scaling Easy 💸: if monitoring is enabled

Example 4. Full image:

Question Answer
Easy to collect artifacts? Hard 🧱to get full image and also there is a need to parse it. But reliable.
Easy to connect remotely? Medium 🍯: depends
Identify relevant data? Medium 🍯: depends on the parser tool.
Pivoting Medium 🍯: depends on the parser tool.
Scaling Medium 🍯: depends on aquisition tools.

Example 5. Cyber Triage:

Question Answer
Easy to collect artifacts? Easy 💸: one exe for local or remote.
Easy to connect remotely? Easy 💸: easy to send over network.
Identify relevant data? Medium 🍯: scores data.
Pivoting Easy 💸: navigation via UI.
Scaling Multiple

Collection

Powershell is the best tool to automate and scale evidence collection accross the enterprise.

Kansa

Not only for collection, but also for analysis (stacking). Can be scaled. Good for acquisition across enterprise.

PowerShell (CSIRT-Collect.ps1)

Two directories: one with Winpmem and 7zip, another with Kape, copied from local install. Makes a RAM image, zips it and sends to the network share. Cleans up the host under investigation after that. A very useful thing: it’ll capture OS build, needed for vol2.py for profiling.

Then, it launches KAPE which performs triage.

KAPE (Kroll Artifact Parser & Extractor)

Seems to be very close to what Kansa frawemork does. What’s the difference? Kansa for live response and hunt; KAPE for forensics. Also, it seems that Kansa is dying. Better explained here by sprint_ska, thanks man. Also, Kansa relies on Windows built-in tools while Kape doesn’t. Kansa scales better.

kape --tsource C: --target RegistryHives --tdest "\results" 
kape --tsource H --target !ALL --tdest "dir" --debug  
kape --msource “dir” --module LECmd --mdest "dir" --trace --debug

Native Hashing Tools

Windows. certutil -file [filename] -hash [md5, sha1, sha256 etc].

macOS. md5, shasum.

Linux. md5sum, sha256sum etc.
