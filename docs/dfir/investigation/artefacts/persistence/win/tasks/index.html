Scheduled Tasks


There are several ways to stay persistent on a Windows machine. One of them is to set up the program to be automatically started at a certain time (most often - at system startup).

C: \Windows\ Tasks\ \*.job
C: \Windows\SchedLgU.txt
C: \Windows\system32\Tasks

at.exe (deprecated but can still be used) and schtasks.exe. For at see at*.job and Schdlgu.txt and Task Scheduler and Security Logs for the second one. WinXP: at jobs run with SYSTEM privileges. Files are created here: \Windows\Tasks and \Windows\System32\Tasks (xml duplicate Win7+). Created with: at.exe 22:22:22 C:\mal.exe or schtasks.exe /create /sc daily /tn winsvchost /tr C:\mal.exe.

Process

  1. Run autorunc.exe on the machines in question (collection), use csv format as output.
  2. Open each file (if there are not too much) with TimeLine Explorer
    1. Filter for untrusted or missing vendors. Such vendors as Google, Firefox might be used in an attempt to trick the user. It would look like (Not verified) Firefx.
    2. Show those enabled ones
    3. Look at the image path, use FindEvil SANS poster as a reference for known bad
    4. Look at the hashes and check against known-good.
    5. Check with supernova, google Publishes and Description
    6. Frequency analysis. What stands out? (see the next sections). Find possible suspicious or obviously malicious things.
    7. Check the triage files from other machines for the same IoCs (Select-String, grep etc).
  3. Stacking (frequency-based outlier analysis)
  4. Frequency analysis. What stands out? (see the next sections)

🗒 If the file doesn’t have an image path (File not found), it’s likely it was moved/deleted and thus is not an active threat.

Suspicious Things (Windows)

  • Known non-system executables like OneDrive in System32 directory.
  • Files with completely meaningless names and usually 1 letter or 1 number like 1.bat or 2.exe or a.exe.
  • Be especially careful with drivers since they have the most power in the PC
  • WMI entries are also worth checking
  • Executables and scripts in a temp directory

Option -a tells autorun to collect ALL. But what is all?

  • Boot
  • Appinit Dll
  • Explorer addons
  • Sidebar gadget
  • Image hijacks
  • IE addons
  • Known dlls
  • Logon startup
  • WMI
  • Office addons
  • Printer mon dlls
  • LSA security providers
  • Autostart services and enabled drivers
  • winlogon entries
  • scheduled tasks
  • winlock protocol and netwrok providers
