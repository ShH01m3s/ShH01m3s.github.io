🤔 How Do I Audit Management Tools?

Created: 26.06.2023

Admins on payroll and those “magnanimous volunteers” often tread the very same path, albeit guided by different compasses of motivation. Once our cunning adversary ascends to the lofty heights of respect usually reserved for the admin on payroll, the misuse of these management tools becomes bound only by the limits of their mischievous imagination.

Windows

Services

❗️Doesn’t require admin privileges to pull off.

📕 sc \\host create service_name binpath="path/to/service"
🏺 Artefact 🚰 Source 🎯 Target
🗄️ Registry ShimCache, BAM/DAM and AmCache for sc.exe traces 🔑 CurrentControlSet\ServicesShimCache and AmCache
📑 File system Prefetch Image file of the exePrefetch
🪵 Event logs 🛡️ 4624 type 3 (sIP, sname), 4697 (service installed) ⚙️ 7034 (service terminated unexpectedly), 7035 (start/stop request sent), 7036 (service started/stopped), 7040 (start type changed), 7045 (service installed)

Tasks

📕 at \\host time "path\to\exe"
📕 schtasks /CREATE /TN taskname /TR C:\path\to\evil.exe /SC once /RU "SYSTEM" /ST 15:00 /S host /u uname

# RU - run as user
# TR - what to do
# S - remote system to connect to

For the below artefacts watch out for at.exe and schtasks.exe.

🛡️Security and 🥀 Task logs

🏺 Artefact 🚰 Source 🎯 Target
🪵 Event logs 4648 (alternate creds) 🛡️ 4698 (created), 4702 (updated), 4699 (deleted), 4700 (enabled), 4701 (disabled), 🥀 106 (created), 140 (updated), 141 (deleted), 200 (executed), 201 (completed)
🗄️ Registry ShimCache, BAM/DAM 🔑 Microsoft\Windows NT\CurrentVersion\Schedule\TasksCache\TasksShimCache, AmCache
📑 File System Prefetch Prefetch📂 C:\Windows\System32\Tasks (newer) and 📂 C:\Windows\Tasks containing .job (binary) or .xml files

WMI

✍🏻 T1047 MITRE ATTACK

This tool’s artefacts usually reside mostly in RAM. Also, it uses WBEM protocol, and thus, it’s tricky to eavesdrop on this connection. If no WinRM - network forensics is an option.

✍🏻 CIM - common info model, shows object-relationships information.

📕 wmic /node:host /user uname process call create "path\to\exe"
📕 Invoke-WmiMethod -Computer host -Class Win32_Process -Name create -Argument "better\be\a\good\argument.exe"

Look out for the following executables wmic being invoked (on the source), along with srccons, mofcomp, wmiprvse and some custom names limited to the attacker’s quirky whims (on the target).

🛡️Security, 🍉 Microsoft-Windows-WMI-Activity%4Operational

🏺 Artefact 🚰 Source 🎯 Target
🪵 Event logs 🛡️ 4648 (alternate creds) 🛡️4624 type 3, 4672 (admin)
🗄️ Registry ShimCache, BAM/DAM, AmCache 🍉 5857 (start or stop, path to provider DLL), 5860/5861 (temp/permanent Event Consumer created), ShimCache, AmCache, Prefetch
📑 File System Prefetch mof files, unauthorised changes to C:\Windows\System32\wbem\Repository

PowerShell Remote

✍️ Close to SSH, traffic is also encrypted.

📕 Enter-PSSession -ComputerName host -Credential uname
📕 Invoke-Command -ComputerName host -ScriptBlock { Start-Process \path\to\process}

🪵 Logs: 🛡️ Security, 🍌 Microsoft-Windows-PowerShell and 🥝 Microsoft-Windows-WinRM.

⚙️ Processes: powershell.exe

🏺 Artefact 🚰 Source 🎯 Target
🪵 Event logs 🛡️ 4648🥝 6 -WSMan Session init (dhost, dIP, c_user, timestamp), 8, 15, 16, 33 - deinit 🍌 40691, 40692, 8193, 8194, 8197
🗄️ Registry ShimCache, BAM/DAM, AmCache
📑 File System Prefetch

Registry

❗️ Requires authentication.

📕 reg add \some\reg\key /v data /t REG_SZ /d "path\to\whatever"

📕 winrs -r:host -u:user command_to_run
