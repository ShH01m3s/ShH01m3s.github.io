3D Secure

domain: general

doctype: article


network , protocol , 3ds

This article is about 3DS protocol (XML-based) which is used to secure online payments. There are three parties in this process: merchant, aquirer and the bank (the issuer of the card). It’s an additional authentication step. Invented by VISA.

Commercial names:

  • Verified by Visa (Visa)
  • SecureCode (MasterCard)
  • J/Secure (JCB)
  • SafeKey (American Express)
  • ProtectBuy (Diners/Discover)

Non 3DS

Liability shift. If there is fraud, the merchant is to be blamed. He’s monitoring and detecting frad without 3DS.

There are three domains:

3DS Domains

  1. Issuer.
    1. client’s browser + app
    2. Enrolment server
    3. access control server
    4. AAV Server
  2. Acquirer.
    1. MPI
    2. Signature Validation Server
  3. Interoperability.
    1. Dir Server
    2. Cert Auth
    3. MC Auth History Server
    4. Attempts processing server
