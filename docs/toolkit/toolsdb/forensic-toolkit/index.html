Autopsy

There is a good course from Basis Technology about there own tool, of course, and I am sure there are lot’s of other tutorials out there available for free, so, this is not meant to be complete user guide. These are things that I have found notable or important to take into account when analysing a system.

Correlation Engine

This module gets data from other modules:

  • hash values from Hash Lookup IM (ingest module)
  • Email addresses (probably, from keyword search)
  • SSID (where does it get them from?)
  • Central Repo (to check for previous occurrences)

So keep this in mind when using it. If you don’t run some ingest module, CE won’t have this info in its disposal.

Registry Analysis

  1. Queries for hives that are stored in System32\config
    1. System
    2. Software
    3. Security
    4. SAM - containing user acc info
  2. Quieries for NTUSER.dat hives
  3. Runs a RegRipper profile

Updates

APFS Disk images support, XRY Text File Reports. Chromium support. More Windows artifacts. New Picture Analyzer (former EXIF Module) + converts HEIC to JPEG and adds as children.

â“ what’s HEIC?

Flags files based on name and path. For example, googlesync.exe goes under category Cloud Storage. Contribute rules. Incorprorating scoring from CyberTriage.

Central Repository. Enabled by default. Doesn’t correlate by default.

Drone analyser. New, based on DHS S&T Contact (DAtCon to parse DJI drone data and creates track points for where the drone was). GPX module creates track points, way points, etc from non-drones. Can be displayed in the new Map viewer.

iOS Analyzer! Finally! Uses iLEAPP. .tar as input file. 4.18 should support disk images, more artifacts and aLEAPP.

New Context Viewer. Where did it come from (downloads, attachments)? When or where was it used (MRU keys, Prefetch, etc)?

Data Source Summary. About this data source, about user activity â—.

Discovery. Search navigation. Items are organized in groups. See copies of the artifacts.

Maps. Geo from iOS, Android, Drone, EXIF.Uses Bing or OpenStreetMap.

Interesting modules:

  1. W10-FaceMesanger@Autopsy is Data Source Module. SQLlite DB can be queried to retrieve deleted items â—Undark. For Win10.
  2. LabCIF DOI - Detection of Objects in Images. YOLOv3 - You only look once. Uses Darknet in GPU mode (if CUDA available) or OpenCV in CPU mode for best performance. https://github.com/labcif/doi.
  3. Camera Fingerprint PRNU. To determine that the photo was taken by the suspect camera. Uses photo response non-uniformity of camera sensor: photo - noise = PRNU. Physical analysis of camera sensor + PRNU. Each sensor pixel is unique. Creates a description of camera sensor (fingerprint). To create a fingerprint, at least 30 photos are needed (white surface such as walls and clouds). Add evidence photos to data source. Select a folder with reference photos in ingest config. If the suspect photo was edited or resized, the information might be inaccurate (check tampering with other tools).
  4. SpeechToText. Type identifiaction module should be run before using this.
  5. BlackVue Geolocation.
  6. Android Usagestats. XML with user interatcion, moving apps to fore- and background and config changes. data/system/usagestats, protobuf files.
  7. TikTok for Android Analyzer. Contacts and Messages. tiktok.py. Move it to InternalPythonModules/android. Open module.py and import the tiptok.py and add the analyzer to the exisiting list.
  8. Forensic Analysis for Mobile Apps (FAMA). Extract user data from a connected Android device (using adb). Produce detailed reports (maps, timelines,multimedia). Recover rows from SQLite (Undark and SQL Deleted Records Parser). Https://github.com/labcif/FAMA. Clone repo, copy contents to autopsy/python_modules. Select live extraction with ADB.
  9. Notification analyser. Windows 10 Notification with “Your Phone” integration. %LOCALAPPDATA%\Microsoft\Windows\Notifications. SQLite3 DB produces WAL (Write Ahead Log) and SHM files. Notifications live for 3 days. Deleted rows can be recovered. YPA (anothe Autopsy plugin - https://github.com/labcif/ypa).
  10. Tinder for Android. Contacts and messages. tinder.py.
