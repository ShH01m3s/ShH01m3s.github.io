radare2

Host platforms: 🍏, 🐧, 🌈 ,📱

Target platforms: 🍏, 🐧, 🌈, 📱

Absoultely free that’s why it’s such a darling for many people. However its a command line tool with lots of commands to keep in mind which might make the analysis slower if you are not using it pretty oftern. Also, I am not sure whether they have some scripting functionality included. One of the advantages is that it has several additional tools in the package (like rabin). The main advantage is that it can be install on ANY platform (mobile phones or PCs, Macs, Windows, Linux). You can statically examine, for example, an ARM binary on a PC for free, but you need to install radare on some ARM device to debug it.

Install

git clone https://github.com/radare/radare2.git
cd radare2
./sys/install.sh

Update

./sys/install.sh

Uninstall

make uninstall
make purge

Procmon

This tools captures almoust all Win API calls (see Possible issues below for exceptions) while being run. When using it, take advantage of the filter (separate menu). For example, choose Operation + condition is + API call RegSetValue + what to do then include will show all calls that are named eaxctly RegSetValue.

There are also some additional filters by category (on the top menu bar): Registry, File system, Process Activity, Network turned on by default.

Host platforms: 🌈

Target platforms: 🌈

Install:

Common commands:

Keywords: activity, winapi, logging

Analogues:

Possible issues: Can miss driver activity of user-mode component talking to a rootkit via I/O controls, can miss some GUI calls (SetWindowsHookEx). Works inconsistently across Win versions, that’s why network activity logs are anrealiable.

Process Explorer

This is the tool to examine RAM (i.e. processes run at the moment). It also visualizes the parent-child relations (which program ran which). Services - pink 🐽, processes - blue 🐳, new processes - green 🌿, terminated - red 🍎. Green and red are temp and these highlights are removed shortly after the processes has started (green) or terminated (red). Can show all dls, that were loaded by the process. Handles window shows all files, mutexes, connections opened by this process. The Verify option that checks the developer (signed 🔑) checks the file on disk, not in memory. Therefore, if the malware uses process replacement, it will not be detected by Process Explorer, but in this case the code in memory and code on disk will differ. Using Strings Tab (Image vs Memory radio button) can help identifying this. Can run Dependency Walker on a running process. Search for handle - Find -> Find Handle or Dll.

Host platforms: 🌈

Target platforms: 🌈

Analogues: Process Hacker

Possible issues: Won’t show rootkits and other malware that use hiding functionality. The Verify option that checks the developer (signed 🔑) checks the file on disk, not in memory. Therefore, if the malware uses process replacement, it will not be detected by Process Explorer, but in this case the code in memory and code on disk will differ.
