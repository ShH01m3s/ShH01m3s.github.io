OS fingerprinting

![photo_2021-02-12 10.06.04](images/photo_2021-02-12 10.06.04.jpeg)

nmap scanning can also show the OS version sometimes

For Windows and Linux:

  • Full connect (nmap -sT). Send SYN flag. If SYN/ACK ✅ is received - open. If RST 🚫 - closed. Reply with ACK if open. The most reliable and the noisiest.
  • Stealth or Half-open (nmap -sS). Send SYN flag. Send SYN flag. If SYN/ACK ✅ is received - open. If RST 🚫 - closed. Reply with ACK if open. ☝️Stealthy but can be noticed by IDS. Firewalls won’t notice since no connection was established.
  • IDLE scan or zombie 🧟 scan (nmap -sI). Used to test a machine using another machine. Conclusions are made based on IPID value. Too long to be explained here. See below.
  • UDP scan (nmap -sU). A datagram is sent (zero byte message). If no response is received - open ✅. If ICMP port unreachable (3.3) - closed 🚫 .
  • ACK scan (nmap -sA -P0 10.10.0.25, nmap -sW, hping A). Slow, mostly to avoid IDS. ❗ Only old BSD. Send ACK flag set only:
    • no response ✅ - open, but behind a firewall.
    • RST - no firewall.
      • RST + TTL < 64 or window size != 0 - ✅ the port is open.
      • RST + TTL > 64 or windows size = 0 - 🚫 the port is closed.

For Linux only:

  • Inverse TCP scna (nmap -sF, nmap -sN, hping -F, hping -P, hping -U). ☝️ Win machines will claim all ports are open. Send any of the following flags: FIN/URG/PUSH/NULL. If no response is received - the port is open ✅ . If RST/ACK - 🚫 closed. See RFC 793.
  • Xmas scan (nmap -sX, hping -X). Send all weird flags set: URG, PUSH, FIN. In case no response ✅ is received - the port is open. If RST/ACK - closed. ☝️Windows machine will claim all open.

IDLE Scan

While all other scans are tampering data on Transport layer (TCP or UDP protocols), this one tampers IPID filed of IP header. This value is a random number 💯 , that is incremented with each packet ✉️ sent (no matter who the receiver is). Latest OS versions are patched and therefore are no longer vulnerable. There are several algorithms for incrementing IPID value. Sometimes it was simply incremented by one. Now let’s assume that’s the case.

A malicious user Eva 🥷 is trying to determine whether a good boy Bob 💂 is online, but Bob doesn’t talk to anyone except Alice. Bob 💂is suspicious and doesn’t trust anyone, but Alice 👩‍🔬, on the other hand, is very careless and never suspects a thing. She’s busy making a huge break-through in chemistry. Because of this carelessness, Eva 🥷 decides to probe Bob 💂 via Alice’s 👩‍🔬 PC. Here’s how she’s doing it:

  1. Eve 🥷 sends Alice 👩‍🔬 a TCP segment with both SYN/ACK flags set.
  2. Since SYN/ACK are usually the second step of TCP handshake and Alice 👩‍🔬 did not initiate any conversation, she responds with RST (basically means “WTF? Go to hell, I’m busy."). In IP header of this packet the IPID value is 12345.
  3. Now Eve 🥷 sends Bob 💂 a packet with only SYN flag. This is usually how a TCP connection starts. She spoofs Alice’s IP. Next, there are two possible scenarios based on whether Bob is online:
    1. If Bob 💂 is online ✅, here is what will happen. Remember, Bob 💂only talks to Alice 👩‍🔬. But since Alice’s 👩‍🔬 IP was spoofed by Eve 🥷, he responds to Alice 👩‍🔬. Eve 🥷 doesn’t see this response, of course. But she doesn’t need to 😊.
      1. Alice knows nothing about her IP being spoofed and sends another “WTF?” (IP packer with RST flag set in TCP header) this time to Bob. Her IPID is incremented by 1 and now is 1+12345 which is 12346.
      2. Now. Eve 🥷 repeats the step #1, i.e. sends Alice 👩‍🔬 a TCP segment with both SYN/ACK flags set.
      3. Poor Alice is tired of being disturbed and replies with another “WTF?” (IP packer with RST flag set in TCP header) to Eve 🥷. Her IPID is inceremented by 1 again and now it’s equal to 1+12346 which is 12347.
      4. Eve 🥷 observes this IPID and concludes that inbetween her first request to Alice and her second request to Alice, she talked to someone else. That’s how she knows that Bob is online. Of course some innocent bystander can spoil this beatuful scheme by talking to Alice as well, but since Eve is doing this very quicly, the chances are not that high if it’s now a server of course.
    2. If Bob 💂 is not online 🚫, here is what will happen. Since he’s not online, he doesn’t reply to Alice.
      1. Since Alice didn’t talk to Bob, her IPID remains 12345.
      2. Now. Eve 🥷 repeats the step #1, i.e. sends Alice 👩‍🔬 a TCP segment with both SYN/ACK flags set again.
      3. Alice 👩‍🔬 replies with another “WTF?” (IP packer with RST flag set in TCP header) to Eve 🥷. Her IPID is inceremented by 1 again and now it’s equal to 1+12345 which is 12346.
      4. Eve 🥷 observes this IPID and concludes that inbetween her first request to Alice and her second request to Alice, she didn’t talked to anyone. That’s how she knows that Bob is not online.

To conclude, if Alice’s 👩‍🔬 IPID has incremented more that by one - Bob’s 💂 most likely online. Otherwise - he’s not online.
